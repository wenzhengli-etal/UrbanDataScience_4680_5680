1. TextBlob and Tokenization¶

TextBlob is a Python library for processing textual data. We need to install TextBlob in command prompt (or terminal) using

pip install textblob

In addition, we need to execute the following command to download the NLTK corpora used by TextBlob.

python -m textblob.download_corpora

Once installation completes, we import from the library the fundamental class for natural language processing (NLP) --- TextBlob.

1.3 Parts-of-Speech (POS) Tagging¶

There are 8 primary English parts of speech:

* noun 
* pronoun  
* verb
* adjective
* adverb
* preposition
* conjunction
* interjection

Many words have multiple meanings. For example, "design" can be a noun or a verb. Determining a word's meaning in the sentence is very important for computers to undertand natural languages. POS tagging is for this purpose.

2. Sentiment Analysis¶

Sentiment analysis is to determine whether text is positive, neutral, or negative. Computers should be able to tell that "the weather is not bad" has a positive sentiment. This is a complex machine learning task. TextBlob provides pretrained machine learning models for us to perform sentiment analysis.

2.1 Default sentiment analyzer¶

By default, TextBlob uses a PatternAnalyzer. We can simply use .sentiment on a TextBlob or a Sentence object to get a Sentiment object.

In [15]:
blob
Out[15]:
TextBlob("the presence of buildings around a park is important in design. They enclose it. They make a definite shape out of the space,so that it appears as an important event in the city scene, a positive feature, rather than a no-account leftover.")
In [16]:
blob.sentiment
Out[16]:
Sentiment(polarity=0.25681818181818183, subjectivity=0.7613636363636364)
  • "polarity" indicates sentiment with a value from -1 (negative) to 1 (positive) with 0 being neutral.
  • "subjectivity" is a value from 0 (objective) to 1 (subjective).

So the text's overall sentiment is positive, and the text is mostly subjective. We can also check sentiment for each sentence. Ask yourself whether each sentence's result makes sense.

In [17]:
for s in range(len(blob.sentences)):

2.2 Load JSON file¶

So far, we know how to read Excel, csv files using pandas, and shapefiles using GeoPandas. But there are many other common types of files. For example, txt file and JSON file, which are the conventional choices for text data. Today, we first learn how to read JSON files in Python.

To read a JSON file, we need two Python functions: open() and json.load().

The first open() is a Python built-in function to open a file and return a file object. This function takes two main arguments, the name of the file and and the mode (listed below) for which we would like to open the file. Sometimes, we also need to specify the "encoding" for special characters.

  • 'r' open for reading (default)
  • 'w' open for writing, truncating the file first
  • 'x' create a new file and open it for writing
  • 'a' open for writing, appending to the end of the file if it exists
  • 'b' binary mode
  • 't' text mode (default)
  • '+' open a disk file for updating (reading and writing)
  • 'U' universal newline mode (deprecated)

The second is to use a Python built-in library called json, which is used to work with JSON data, and its function called .load() to parse the opened JSON file into a Python object (most of time would be a dictionary)

We use with...as...: structure to use open() and json.load() together.
